Socket Chat + End-to-End Encryption Demo

서버는 전달만 하고, 내용은 브라우저끼리만 봅니다.

입장할 때 공개키를 교환하고, 메시지는 각 브라우저에서 AES-GCM으로 암호화한 뒤 WebSocket 서버가 암호문만 중계합니다.

연결 전 아직 입장하지 않음 실제 서비스라면 사용자 인증과 공개키 검증이 추가로 필요합니다.
1. 공개키 교환 브라우저끼리 공개키만 공유
2. 공유키 생성 SwiftSeal-v1 HKDF 키 파생
3. 메시지 암호화 AES-GCM + AAD
4. 서버 중계 서버는 암호문만 확인
5. 복호화 상대 브라우저에서만 평문 복원

입장

키 교환 상태

  • 아직 연결된 상대가 없습니다.

SwiftSeal-v1 프로토콜

join 공개키 등록 HKDF 공유 비밀에서 메시지 키 생성 AES-GCM AAD로 메타데이터 변조 방지 encrypted-message 암호문 중계

암호화 채팅

브라우저 창을 2개 열고 같은 방에 입장하면 테스트할 수 있습니다.

방에 입장하면 여기서 메시지를 주고받을 수 있습니다.

서버가 보는 내용

서버는 알고리즘 이름, keyId, 송수신자, 암호문, IV만 볼 수 있습니다. 평문과 공유키는 서버로 전송되지 않습니다.

아직 중계된 암호문이 없습니다.
